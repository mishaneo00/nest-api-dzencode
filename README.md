# Nested Comments System (NestJS)

Полнофункциональный бэкенд для системы комментариев с бесконечной вложенностью, авторизацией, загрузкой файлов и Real-time обновлением.

## 🛠 Стек технологий

- **Framework:** NestJS (HTTP), TypeScript
- **Database:** PostgreSQL (TypeORM)
- **Real-time:** WebSockets (Socket.io)
- **Task Queue:** BullMQ (Redis)

## API Endpoints

### Авторизация и Безопасность (`/auth`)

#### **POST** `/session`

Создание анонимной сессии для пользователя.

- **Описание:** Генерирует временный JWT-токен с уникальным `fingerprint`, который необходим для идентификации пользователя без классической регистрации.
- **Ответ:** JSON объект `TokenResponseDto` -> {access_token}.

#### **POST** `/captcha`

Генерация новой капчи для защиты от спама.

- **Описание:** Создает SVG-изображение капчи и сохраняет правильный ответ в Redis с ограниченным TTL (5 минут).
- **Заголовки:** `Cache-Control: no-store` (гарантирует получение актуальной капчи без кеширования браузером).
- **Ответ:** ```JSON объект
  {
  "key": "uuid-идентификатор-капчи",
  "svg": "<svg>...</svg>"
  }

### Комментарии (`/comments`)

#### **GET** `/comments`

Получение списка комментариев с поддержкой курсорной пагинации и серверного кеширования.

- **Query параметры:**
  - `limit` (number): количество записей (default: 10).
  - `cursor` (number): ID комментария для отсчета пагинации (ID последнего элемента).
  - `direction` (string): направление `next` (вперед) или `prev` (назад).
- **Особенности:**
  - Используется `JwtOptionalAuthGuard`: если передан токен в Headers, данные будут персонализированы (пометки "мой комментарий").
  - **Кеширование:** Ответ кешируется на стороне сервера (TTL: 30 секунд) через `FirstPageCacheInterceptor`. Кеширование настроено только для первой страницы.

---

#### **POST** `/comments/create`

Создание комментария с поддержкой вложенности и загрузки файлов.

- **Тип запроса:** `multipart/form-data`
- **Заголовки (Headers):**
  - `Authorization`: `Bearer <JWT_TOKEN>` (обязательно)
  - `x-captcha-key`: UUID полученной капчи.
  - `x-captcha-value`: Введенное пользователем текстовое значение капчи.
- **Тело запроса (FormData):**
  - `text` (string): Текст комментария (обязательно, поддерживает разрешенные HTML-теги).
  - `parentId` (number): ID родительского комментария (для реализации вложенности).
  - `homepage` (string): URL домашней страницы пользователя (опционально).
  - `file` (file): Изображение (JPG, PNG, GIF) до 2мб или текстовый файл (TXT) до 100кб.
- **Безопасность:**
  - `CaptchaGuard`: Валидация капчи через кастомные заголовки.
  - `JwtStrongAuthGuard`: Обязательная проверка JWT-токена.
  - `FileSignaturePipe`: Проверка файла на соответствие сигнатуре (Magic Numbers).
- **Механика обработки (HTTP/WS):**
  - **Мгновенный ответ:** Клиент получает созданный комментарий по HTTP сразу. Если есть файл, он приходит со статусом `isLoaded: false`.
  - **Real-time рассылка:** Комментарий мгновенно транслируется всем участникам через WebSocket.
  - **Фоновая обработка:** Файлы (картинки) обрабатываются асинхронно через очередь BullMQ (Redis), файлы TXT загружаются сразу.
  - **Обновление статуса:** После завершения обработки сервер отправляет уведомление по сокету, обновляя статус файла на `isLoaded: true` у всех клиентов без перезагрузки страницы.

---

#### **DELETE** `/comments/:id`

Удаление комментария по его уникальному ID.

- **Параметры:** `id` (число в URL).
- **Заголовки (Headers):**
  - `Authorization`: `Bearer <JWT_TOKEN>`
- **Логика:**
  - Удаление разрешено только владельцу комментария. Проверка осуществляется по `fingerprint` пользователя, извлеченному из JWT.

### Файловый сервис (`/files`)

#### **GET** `/uploads/:filename`

Доступ к загруженным файлам.

- **Параметры:** `filename` в пути и `mime` в Query-строке.
- **Результат:** Стриминг файла для скачивания.

## Real-time (WebSocket / Socket.io)

Сервер транслирует изменения в реальном времени. Названия событий:

1. **`comment:created`** — Новый комментарий (Payload: `CommentResponseDto`).
2. **`comment:file_ready`** — Файл обработан и готов (Payload: `{ fileId, isLoaded: true }`).
3. **`comment:deleted`** — Комментарий удален (Payload: `CommentResponseDto`).

---

## Безопасность и Валидация

- **XSS Protection:** Автоматическая очистка HTML-тегов (разрешены: `a, strong, i, code`).
- **SQL Injection:** Использование TypeORM Query Builder с параметризацией запросов.
- **File Validation:** Проверка сигнатур (Magic Numbers) через `FileSignaturePipe`.
